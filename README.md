# EPC 主合同学习打卡

Masdar Hybrid Project 1 EPC 主合同(DBO1-2006857704 v38, PRE-EXECUTION)的自学打卡工具。纯静态页面,无依赖、无构建、无后端,进度存在浏览器 localStorage 里。

**两条学习线:**

| 线 | 天数 | 每天 | 内容 |
|---|---|---|---|
| 主线 | 42 天 | 60 分钟 | 全合同 41 条,三轮由粗到细:骨架 → 精读 → 应用 |
| 索赔专线 | 21 天 | 30 分钟 | 权利来源 → 程序时效 → 实体举证 → 防守 → 争议实战 |

共 266 道自测题,每题附答案与条款出处。内置四张速查表:保函生命周期、项目时间轴、索赔地图、时效日历与上限。

## 部署到 GitHub Pages

```bash
git init
git add .
git commit -m "EPC study tracker"
git branch -M main
git remote add origin https://github.com/<你的用户名>/<仓库名>.git
git push -u origin main
```

然后在仓库页面 **Settings → Pages → Build and deployment**:
- Source 选 `Deploy from a branch`
- Branch 选 `main`,目录选 `/ (root)`
- Save,等一两分钟

访问 `https://<你的用户名>.github.io/<仓库名>/`。

仓库里的 `.nojekyll` 是必需的,它告诉 GitHub Pages 不要用 Jekyll 处理这些文件。

> 想设成私有仓库的话,GitHub Pages 对私有仓库需要付费方案。免费方案下如果不想公开,直接把 `index.html` 下载到本地双击打开也能用 —— 只是用 `file://` 打开时部分浏览器会禁用 localStorage,进度只保存在当前标签页(页面顶部会有红色提示)。

双击 `index.html` 就能用。想让进度正常保存,起一个本地服务器:

## 怎么改内容

学习内容和代码是分开的,改内容不用碰代码。

```
.
├── index.html              页面骨架
├── .nojekyll               GitHub Pages 必需
├── assets/
│   ├── style.css           样式
│   └── app.js              逻辑(一般不用改)
└── data/
    ├── track-main.js       主线 42 天  ← 改这里
    ├── track-claims.js     索赔专线 21 天  ← 改这里
    └── reference.js        速查表面板  ← 改这里
```

### 改某一天的内容

打开 `data/track-main.js`,找到对应的 `"n": 7`,直接改:

```js
{
  "n": 7,                          // 第几天,别动
  "r": 1,                          // 主线是轮次 1/2/3;索赔线是模块 1~5
  "t": "Clause 20 工期与延误违约金",   // 标题
  "s": [                           // 学习内容,每项是 [栏目名, 内容]
    ["读", "Clause 20 全条;交叉读 Exhibit B Annex 10"],
    ["产出", "DLD 触发—计算—上限—退还 四段式笔记"]
  ],
  "qa": [                          // 自测题
    { "q": "DLD 的两个触发点是什么?", "a": "① …… ② ……" }
  ]
}
```

文本里可以用 `**加粗**`。改完刷新页面即可,如果没变化就强制刷新(Cmd/Ctrl + Shift + R)清一下缓存。

### 加一天

在数组末尾加一条,`n` 用下一个序号,然后到 `assets/app.js` 里把对应 track 的 `segs` 改一下 —— 那是进度带的分段定义:

```js
segs: [[1, 12, '一轮 · 骨架'], [13, 34, '二轮 · 精读'], [35, 42, '三轮 · 应用']]
//      起  止   段标题
```

### 加一张速查表

在 `data/reference.js` 的对象里加一个 key:

```js
"mynew": { "label": "我的表", "html": "<h3>标题</h3><p>内容</p>" }
```

标签页会自动出现,不用改别的地方。可用的 CSS class:`.num`(等宽数字)、`.note`(红色引注)、`.tl`(等宽预格式框,配 `<pre class="tl">`)。

### 改每天的时长

`assets/app.js` 顶部 `TRACKS` 里的 `minutes` 字段,主线 60、索赔线 30。计时器会按这个值归零。

## 用法

- **顶部两个按钮**切换学习线,两条线的进度分开记
- **进度带**就是导航,点哪天跳哪天;琥珀色是今天该做的,深青色是已完成
- **点问题**展开答案,标「答对 / 没答上」自动算正确率
- **计时器**单击开始或暂停,双击归零;超时会变红,超时就停,不补课
- **打卡记录**标签页里可以导出 / 导入进度 JSON,换设备或清缓存前记得导一份

## 两条线怎么排

三种排法,选一种:

- **A. 串行** —— 主线 42 天跑完再跑索赔专线 21 天,共 63 天。适合想先建全局认知的人。
- **B. 交替** —— 主线一轮(Day 1–12)结束后,索赔专线与主线二轮隔天交替,总长约 43 天。推荐这种,索赔的东西在有了骨架之后学吸收最好。
- **C. 并行** —— 每天主线 40 分钟 + 索赔 20 分钟。强度最大,不建议长期。
